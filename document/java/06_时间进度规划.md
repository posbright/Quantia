## 4. 关键路径分析

```
关键路径 = 准备 (1w) 
         → Phase 1 (4w) 
         → Phase 2 (5w) 
         → 评估 (1w)
         → Phase 4 (6w)  [假设 Go]
         → 灰度 (3w)
         ────────────── = 20 周 (关键路径，任何 delay 都影响总进度)

非关键路径 = Phase 3 (ai-decision, 4w) 
          可在 Phase 2 和 Phase 4 之间并行，deadline 要求不严
```

**Slack 分析**:
- Phase 3 可延迟 2-3 周,不影响关键路径
- Phase 1 若 delay 1 周 → 全项目 delay 1 周
- Phase 4 若 delay 2 周 → 全项目 delay 2 周

## 7. 变更管理 & Contingency

### 7.1 延期场景

| 场景 | 影响 | 应对 |
|---|---|---|
| Phase 1 延期 1w | 全项目延期 1w | 增加开发人员 或 并行 Phase 2 某些工作 |
| Phase 2 延期 2w | 全项目延期 2w | 核心功能先发布,非核心延后 |
| 中期评估 No-go | 停止后续,聚焦 Python | 6 个月后重启 OR 转向其他优化 |
| 外聘顾问不可用 | 培训进度 ↓ | 自学 OR 延聘其他顾问 |

### 7.2 加速措施

| 措施 | 成本 | 效果 |
|---|---|---|
| 增加 1 名 Java 开发 | +人力成本 | Phase 1/2 加速 1-2 周 |
| 取消 Phase 3,直接上 Phase 4 | 功能不完整 | 节省 4 周,但交易风险高 |
| 并行多个 Phase | 管理复杂 | 最多节省 2 周 |
