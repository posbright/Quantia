#### 2.1.1 Spring Cloud Gateway 配置

**职责**:
- HTTP 请求路由(区分 Python web/Java 服务)
- 统一鉴权、认证
- 限流、熔断、重试
- 请求/响应日志、追踪

**路由规则**:

### 2.2 应用服务层

#### 2.2.1 notification-svc (第一个 Java 微服务)

**架构**:
```
RabbitMQ Topic: notification.events
  ↓
[notification-svc Consumer]
  • ConnectionFactory: spring-boot-starter-amqp
  • Listener: @RabbitListener(queues="notification.queue")
  • Processor: 消息转换、模板渲染、渠道分发
  • Channel:
    - DingTalk: webhook 推送
    - Email: 邮件发送
    - SMS: 短信接口 (预留)
  • Retry: exponential backoff (1s, 2s, 4s, 8s, max 3 times)
  • Fallback: 失败后 status='failed', error_message 记库
  ↓
MySQL: UPDATE cn_stock_notification_event 
  SET status='sent', response_json='...', sent_at=NOW()
  WHERE id=?
```

**消息契约**:
```json
{
  "id": "evt_123456",
  "event_type": "paper_trade",
  "channel": "dingtalk",
  "paper_id": 42,
  "trade": {
    "code": "600000",
    "name": "浦发银行",
    "direction": "buy",
    "amount": 100,
    "price": 9.50,
    "trade_date": "2026-06-11",
    "commission": 0.0285,
    "cash_before": 1000000,
    "cash_after": 999050,
    "reason": "keep_increasing 策略命中"
  },
  "timestamp": "2026-06-11T14:30:00Z",
  "retry_count": 0,
  "max_retries": 3
}
```

**Java 实现框架**:
```java
@Service
public class NotificationProcessor {
    
    @RabbitListener(queues = "notification.queue")
    public void processNotification(NotificationEvent event) throws Exception {
        try {
            NotificationChannelFactory.getChannel(event.getChannel())
                .send(event);
            
            // 成功后 update DB
            notificationRepository.markAsSent(event.getId());
        } catch (Exception e) {
            // 失败后重试(由 RabbitMQ 容器自动处理 + maxRetries)
            throw new AmqpRejectAndDontRequeueException(e);
        }
    }
}
```

#### 2.2.2 im-gateway-svc (第二个 Java 微服务)

**架构**:
```
DingTalk Webhook Callback
  ↓
[im-gateway-svc]
  1. 验证签名 (DingTalk secret)
  2. 白名单检查 (QUANTIA_IM_OPERATOR_WHITELIST)
  3. 风控检查
     - 单笔金额: value <= QUANTIA_IM_MAX_SINGLE_VALUE
     - 日额度: daily_sum <= QUANTIA_IM_MAX_DAILY_VALUE
     - 命令过期: TTL <= QUANTIA_IM_COMMAND_TTL_SECONDS
  4. 指令解析 (confirm_buy / confirm_sell / cancel / adjust)
  5. 落库 to cn_stock_trade_command (status='pending')
  6. 发布到 Kafka: trading.commands
  ↓
Kafka Topic: trading.commands
  ↓
Python job 订阅处理 / Java live-trade-svc 订阅处理
```

**消息契约**:
```json
{
  "command_id": "cmd_654321",
  "channel": "dingtalk",
  "message_id": "msg_789",
  "operator": "user_123",
  "command_type": "confirm_buy",
  "code": "600000",
  "amount": 100,
  "price_limit": 10.0,
  "estimated_value": 1000,
  "wind_control_passed": true,
  "created_at": "2026-06-11T14:30:00Z"
}
```

**风控规则 (Java 中硬编码)**:
```java
@Service
public class RiskControlService {
    
    public RiskCheckResult checkCommand(TradeCommand cmd) {
        // 1. 单笔检查
        if (cmd.getEstimatedValue() > MAX_SINGLE_VALUE) {
            return RiskCheckResult.rejected("单笔金额超限");
        }
        
        // 2. 日额度检查
        LocalDate today = LocalDate.now();
        double dailyUsed = this.getDailyUsedAmount(today);
        if (dailyUsed + cmd.getEstimatedValue() > MAX_DAILY_VALUE) {
            return RiskCheckResult.rejected("日额度超限");
        }
        
        // 3. 白名单检查
        if (!operatorWhitelist.contains(cmd.getOperator())) {
            return RiskCheckResult.rejected("操作人未授权");
        }
        
        return RiskCheckResult.approved();
    }
}
```

#### 2.2.3 ai-decision-svc (可选，P3)

**职责**: 为选股信号提供 AI gate 决策

**架构**:
```
Python job / web handler
  ↓
HTTP POST /api/ai-decision/gate
  {
    "code": "600000",
    "direction": "buy",
    "source": "strategy:keep_increasing",
    "features": {...}
  }
  ↓
[ai-decision-svc]
  1. 读缓存 (Redis, TTL=1h)
  2. 若未缓存,调 OpenAI/Qwen API
  3. 提取评分 (0-100)
  4. 应用阈值:
     - Buy: score >= buy_threshold (e.g. 60) → PASS
     - Sell: score <= sell_threshold (e.g. 40) → PASS
  5. 记库 to cn_stock_ai_score
  6. 返回结果
  ↓
Response:
{
  "score": 72,
  "gate_result": "PASS",
  "model": "gpt-4",
  "latency_ms": 1200,
  "cached": false
}
```

**缓存与降级**:
```java
@Service
public class AiDecisionService {
    
    public AiGateResult evaluateGate(EvaluationRequest req) {
        // 1. Redis 缓存 (prompt hash + input hash)
        String cacheKey = buildCacheKey(req);
        AiGateResult cached = redisTemplate.opsForValue().get(cacheKey);
        if (cached != null) {
            return cached.withCached(true);
        }
        
        // 2. 调 LLM (with circuit breaker)
        AiGateResult result = null;
        try {
            result = this.callLLM(req);
        } catch (LLMTimeoutException e) {
            // 降级: 返回 NEUTRAL, 不 block 交易
            result = AiGateResult.neutral("AI 服务暂时不可用，自动通过");
        }
        
        // 3. 缓存 1 小时
        redisTemplate.opsForValue().set(cacheKey, result, Duration.ofHours(1));
        
        // 4. 异步记库 (不 block 返回)
        this.asyncSaveScore(req, result);
        
        return result;
    }
}
```

#### 2.2.4 live-trade-svc (高风险，P4)

**职责**: 实盘下单、对账、成交记录

**架构**:
```
Kafka Topic: trading.commands
  ↓
[live-trade-svc Consumer]
  ↓
  1. 幂等检查 (upsert with unique constraint)
     INSERT INTO cn_stock_trade_command_processing (command_id, ...)
     ON DUPLICATE KEY UPDATE ...
  
  2. 二次风控 (金额验证)
  
  3. Broker Adapter 调用
     - DryRun (当前, for 沙箱)
     - GF Client (未来, for 格致客户端)
     - 其他 Broker (可扩展)
  
  4. 成交回调处理
     - 更新 cn_stock_trade_command status='executed'
     - 记录成交日志 to cn_stock_trade_execution_log
     - 发布到 Kafka: execution.completed
  
  5. 对账流程 (T+1)
     - 获取 Broker 成交单
     - 与 DB 对比
     - 差异告警
  ↓
MySQL:
  UPDATE cn_stock_trade_command SET status='executed', ...
  INSERT INTO cn_stock_trade_execution_log ...
```

**关键设计**:
- **幂等性**: command_id 唯一，重复消息自动去重
- **状态机**: pending → processing → executed / rejected
- **对账**:
  ```
  SELECT 成交单 FROM Broker WHERE trade_date = TODAY
  EXCEPT
  SELECT 成交单 FROM cn_stock_trade_execution_log WHERE trade_date = TODAY
  ```
- **失败补偿**: 成交失败后自动重试，超 3 次进入人工审查

**Java 实现**:
```java
@Service
public class TradeExecutionService {
    
    @Transactional(isolation = Isolation.SERIALIZABLE)
    public void executeCommand(TradeCommand cmd) {
        // 1. 插入 processing 记录 (unique constraint on command_id)
        try {
            this.saveProcessingRecord(cmd);
        } catch (DataIntegrityViolationException e) {
            log.info("重复命令, 跳过: {}", cmd.getCommandId());
            return;
        }
        
        // 2. 调 Broker adapter
        BrokerOrderResult result = brokerAdapter.placeOrder(cmd);
        
        // 3. 更新状态
        if (result.isSuccess()) {
            tradeCommandRepo.updateStatus(cmd.getId(), "executed", result.getOrderId());
            executionLogRepo.save(buildExecutionLog(cmd, result));
        } else {
            tradeCommandRepo.updateStatus(cmd.getId(), "rejected", result.getErrorMsg());
        }
    }
    
    @Scheduled(fixedRate = 3600000) // 1 小时对账一次
    public void dailyReconciliation() {
        List<BrokerTrade> brokerTrades = brokerAdapter.getTodayTrades();
        List<ExecutionLog> dbTrades = executionLogRepo.findByTradeDate(LocalDate.now());
        
        Set<String> diff = brokerTrades.stream()
            .map(BrokerTrade::getOrderId)
            .filter(id -> !dbTrades.stream().map(ExecutionLog::getOrderId).collect(Collectors.toSet()).contains(id))
            .collect(Collectors.toSet());
        
        if (!diff.isEmpty()) {
            alerting.sendAlert("交易对账差异: " + diff);
        }
    }
}
```

#### 2.3.3 数据一致性保证

**策略**: 最终一致性 + 对账

```
场景 1: 通知投递成功
- Python job INSERT cn_stock_notification_event (status='pending')
- MQ 消息分发 (at-least-once 语义)
- Java notification-svc 处理成功
- UPDATE status='sent'
- 如果 Java 宕机未 UPDATE: MQ 重试 3 次后失败，status='failed'，告警

场景 2: 交易执行
- Java im-gateway-svc INSERT cn_stock_trade_command (status='pending')
- MQ 分发给 live-trade-svc
- Java live-trade-svc 调 Broker
- Broker 返回 order_id
- UPDATE status='executed'
- 若发生分裂: T+1 日志对账，检查 Broker 中是否有该 order_id

场景 3: 配置变更
- Python web handler SAVE 参数到 cn_strategy_params
- Kafka 发布 'strategy.config.changed' 事件
- Java ai-decision-svc 订阅，刷新本地缓存
- 同时 Python job 定时读最新参数(polling as fallback)
```

#### 2.4.1 RabbitMQ (通知可靠投递)

**拓扑**:

```
│  (Topic, persistent)      │
│                            │
├─ Topic: notification.*    ├─> Queue: notification.queue
│                            │  (Durable, TTL=48h)
└────────────────────────────┘
       ▲
       │ AMQP Producer
       │
Python job / web handler
```

**Exchange 配置**:
```yaml
spring:
  rabbitmq:
    host: ${RABBITMQ_HOST}
    port: 5672
    username: ${RABBITMQ_USER}
    password: ${RABBITMQ_PASSWORD}

    listener:
      simple:
        acknowledge-mode: MANUAL  # 手动确认
        concurrency: 10
        max-concurrency: 20
        prefetch: 1               # 预取数，避免某个消费者过载
        default-requeue-rejected: false  # 拒绝后进入 DLQ

      exchange: notification
      queue: notification.queue
      routing-key: 'notification.*'
      dlx: notification-dlx
      dlq: notification-dlq
      ttl: 172800000  # 48 小时
```

**重试策略**:
```java
@Configuration
public class RabbitConfig {
    
    @Bean
    public RabbitTemplate rabbitTemplate(ConnectionFactory connectionFactory) {
        RabbitTemplate template = new RabbitTemplate(connectionFactory);
        template.setMessageConverter(jackson2JsonMessageConverter());
        
        // 发布者确认 (Publisher Confirms)
        template.setConfirmCallback((correlationData, ack, cause) -> {
            if (!ack) {
                log.error("消息发布失败: {}, {}", correlationData.getId(), cause);
            }
        });
        
        // 发布者返回 (Publisher Returns)
        template.setMandatory(true);
        template.setReturnsCallback(returned -> {
            log.error("消息路由失败: {}", returned.getMessage());
        });
        
        return template;
    }
    
    @Bean
    public Queue notificationQueue() {
        return QueueBuilder.durable("notification.queue")
            .ttl(172800000)  // 48 小时过期
            .deadLetterExchange("notification-dlx")
            .deadLetterRoutingKey("notification.dlq")
            .build();
    }
}
```

#### 2.4.2 Kafka (交易事件流)

**主题设计**:
```
Topic: trading.commands
  Partitions: 3 (按 code % 3 分片，保证同股票顺序性)
  Replication: 2
  Retention: 7 天
  
  Message Key: code (保证同股票消息顺序)
  Message Value: TradeCommand JSON

Topic: trading.executions
  Partitions: 3
  Replication: 2
  Message Key: order_id
  Message Value: ExecutionResult JSON
```

**生产者配置** (Python 端):
```python
# quantia/job/strategy_data_daily_job.py
from kafka import KafkaProducer
import json

producer = KafkaProducer(
    bootstrap_servers=['kafka:9092'],
    value_serializer=lambda v: json.dumps(v).encode('utf-8'),
    acks='all',  # 所有副本确认
    retries=3,
    compression_type='gzip'
)

# 发布交易命令
def publish_trade_command(cmd):
    producer.send(
        'trading.commands',
        key=cmd['code'].encode(),
        value=cmd
    )
```

**消费者配置** (Java 端):
```java
@Configuration
public class KafkaConfig {
    
    @Bean
    public ConsumerFactory<String, TradeCommand> consumerFactory() {
        Map<String, Object> props = new HashMap<>();
        props.put(ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, kafkaServers);
        props.put(ConsumerConfig.GROUP_ID_CONFIG, "live-trade-consumer-group");
        props.put(ConsumerConfig.ENABLE_AUTO_COMMIT_CONFIG, false);  // 手动提交
        props.put(ConsumerConfig.AUTO_OFFSET_RESET_CONFIG, "earliest");
        props.put(ConsumerConfig.MAX_POLL_RECORDS_CONFIG, 100);
        
        return new DefaultKafkaConsumerFactory<>(props, 
            new StringDeserializer(), 
            new JsonDeserializer<>(TradeCommand.class));
    }
    
    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, TradeCommand> 
    kafkaListenerContainerFactory() {
        ConcurrentKafkaListenerContainerFactory<String, TradeCommand> factory =
            new ConcurrentKafkaListenerContainerFactory<>();
        factory.setCommonErrorHandler(new DefaultErrorHandler());
        factory.setConcurrency(5);
        return factory;
    }
}

@Service
public class TradeCommandListener {
    
    @KafkaListener(topics = "trading.commands", groupId = "live-trade-group")
    public void onTradeCommand(TradeCommand cmd, Acknowledgment ack) {
        try {
            tradeExecutionService.execute(cmd);
            ack.acknowledge();  // 手动提交 offset
        } catch (Exception e) {
            log.error("处理交易命令失败", e);
            // 不提交 ack，消息会被重试
        }
    }
}
```
