## 1. 改造总体流程

```
准备阶段 (1 周)
    ↓ 环境搭建、团队培训、代码框架
Phase 1: notification-svc (4 周) ← 最低风险学习机制
    ↓ 验证 MQ + 跨语言通信
Phase 2: im-gateway-svc (5 周)
    ↓ 增加风控、追踪、整合测试
Phase 3: 中期评估 (1 周)
    ↓ 决定是否继续 P3/P4

    ↓ LLM 集成、缓存、降级
Phase 4: live-trade-svc (谨慎，6 周)
    ↓ 最高风险，严格对账、金融级测试
验收阶段 (2 周)

总周期: 24 周 (6 个月)
```

   ```java
   @Configuration
   public class RabbitConfig {
       
       public static final String NOTIFICATION_EXCHANGE = "notification";
       public static final String NOTIFICATION_QUEUE = "notification.queue";
       public static final String NOTIFICATION_ROUTING_KEY = "notification.*";
       
       @Bean
       public TopicExchange notificationExchange() {
           return new TopicExchange(NOTIFICATION_EXCHANGE, true, false);
       }
       
       @Bean
       public Queue notificationQueue() {
           return QueueBuilder.durable(NOTIFICATION_QUEUE)
               .ttl(172800000)  // 48 小时
               .deadLetterExchange("notification-dlx")
               .build();
       }
       
       @Bean
       public Binding notificationBinding(
           Queue queue, TopicExchange exchange) {
           return BindingBuilder.bind(queue)
               .to(exchange)
               .with(NOTIFICATION_ROUTING_KEY);
       }
   }
   ```

4. **消息监听器** (service/NotificationProcessor.java)
   ```java
   @Service
   @Slf4j
   public class NotificationProcessor {
       
       @RabbitListener(queues = RabbitConfig.NOTIFICATION_QUEUE)
       public void process(NotificationEvent event) throws Exception {
           try {
               log.info("处理通知事件: {}", event.getId());
               
               NotificationChannel channel = 
                   channelFactory.getChannel(event.getChannel());
               channel.send(event);
               
               // 更新状态为 sent
               notificationRepository.markAsSent(event.getId());
               
               // 记录指标
               metrics.recordSent(event.getChannel());
               
           } catch (Exception e) {
               log.error("通知发送失败", e);
               metrics.recordFailed(event.getChannel());
               throw new AmqpRejectAndDontRequeueException(e);
           }
       }
   }
   ```

   ```java
   public class DingTalkChannel implements NotificationChannel {
       
       @Override
       public void send(NotificationEvent event) throws Exception {
           String webhook = System.getenv("QUANTIA_DINGTALK_WEBHOOK");
           String markdown = buildMarkdown(event);
           
           HttpRequest request = HttpRequest.newBuilder()
               .uri(URI.create(webhook))
               .POST(HttpRequest.BodyPublishers.ofString(
                   "{\"msgtype\": \"markdown\", \"markdown\": {\"text\": \"" + markdown + "\"}}"
               ))
               .build();
           
           HttpResponse<String> response = 
               HttpClient.newHttpClient().send(request, 
                   HttpResponse.BodyHandlers.ofString());
           
           if (response.statusCode() != 200) {
               throw new DingTalkException("钉钉推送失败: " + response.statusCode());
           }
       }
   }
   ```

3. **Python 适配器** (quantia/lib/notification_publisher.py)
   ```python
   import pika
   import json
   import datetime
   
   class NotificationPublisher:
       def __init__(self, rabbitmq_host='localhost'):
           self.connection = pika.BlockingConnection(
               pika.ConnectionParameters(rabbitmq_host))
           self.channel = self.connection.channel()
           self.channel.exchange_declare(
               exchange='notification',
               exchange_type='topic',
               durable=True)
       
       def send(self, event):
           """发布通知事件到 RabbitMQ"""
           msg = {
               'id': str(uuid.uuid4()),
               'event_type': event['event_type'],
               'channel': event['channel'],
               'paper_id': event.get('paper_id'),
               'trade': event.get('trade'),
               'timestamp': datetime.datetime.now().isoformat(),
               'retry_count': 0,
               'max_retries': 3
           }
           
           self.channel.basic_publish(
               exchange='notification',
               routing_key=f"notification.{event['event_type']}",
               body=json.dumps(msg),
               properties=pika.BasicProperties(
                   content_type='application/json',
                   delivery_mode=2  # persistent
               ))
   ```

1. **重试服务**:
   ```java
   @Service
   @Slf4j
   public class RetryService {
       
       @Scheduled(fixedRate = 60000)  // 每分钟检查一次
       public void retryPendingNotifications() {
           List<NotificationEvent> events = repo.findByStatusAndNextRetryAtBefore(
               "pending", LocalDateTime.now());
           
           for (NotificationEvent event : events) {
               if (event.getRetryCount() >= event.getMaxRetries()) {
                   log.warn("通知已达重试上限: {}", event.getId());
                   event.setStatus("failed");
                   repo.save(event);
                   alerting.sendAlert("通知失败需人工介入: " + event.getId());
                   continue;
               }
               
               try {
                   processor.process(event);
               } catch (Exception e) {
                   event.setRetryCount(event.getRetryCount() + 1);
                   long delaySeconds = Math.min(Math.pow(2, event.getRetryCount()), 3600);
                   event.setNextRetryAt(LocalDateTime.now().plusSeconds((long) delaySeconds));
                   repo.save(event);
               }
           }
       }
   }
   ```

2. **DLQ 处理**:
   ```java
   @Configuration
   public class DeadLetterConfig {
       
       @Bean
       public Queue dlq() {
           return QueueBuilder.durable("notification-dlq").build();

       @Bean
       public TopicExchange dlx() {
           return new TopicExchange("notification-dlx", true, false);
       }
       
       @Bean
       public Binding dlqBinding(Queue dlq, TopicExchange dlx) {
           return BindingBuilder.bind(dlq).to(dlx).with("*");
       }

   @Service
   @Slf4j
   public class DeadLetterHandler {
       
       @RabbitListener(queues = "notification-dlq")
       public void handleDLQ(NotificationEvent event) {

           alerting.sendAlert("通知 DLQ 告警: " + event.getId());
       }
   }
   ```

## 9. 风险与缓解(简要)

| 风险 | 影响 | 缓解 |
|---|---|---|
| RabbitMQ 宕机 | 通知不发 | 本地库缓存 + MySQL 作备份队列 |
| Kafka 分区不均 | 交易延迟 | Rebalance 策略 + 监控 lag |
| Python-Java 网络分裂 | 两端数据不一致 | 每日对账 + 告警 |
| 团队学习曲线陡 | 交付延期 | 持续培训 + Code review |
