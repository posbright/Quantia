# Quantia Java 微服务改造 — 单机 8GB 精简部署实操模板

**文档版本**: 1.0  
**日期**: 2026-06-11  
**目标**: 用最小成本在单机 8GB 上快速上线，并可平滑扩容。

---

## 1. 使用文件

- Compose: `document/java/single-node-8gb/docker-compose.single8gb.yml`
- Env 模板: `document/java/single-node-8gb/.env.example`
- Prometheus 精简配置: `document/java/single-node-8gb/prometheus.single8gb.yml`

---

## 2. 部署步骤 (15 分钟)

### 2.1 准备

```bash
cd document/java/single-node-8gb
cp .env.example .env
# 编辑 .env, 至少修改 3 个密码
```

### 2.2 启动核心组件

```bash
# 仅启动核心服务(推荐先不用监控 profile)
docker compose --env-file .env -f docker-compose.single8gb.yml up -d

# 查看状态
docker compose -f docker-compose.single8gb.yml ps
```

### 2.3 可选启动轻量监控

```bash
docker compose --env-file .env -f docker-compose.single8gb.yml --profile obs up -d prometheus node-exporter
```

## 3. 验收清单 (上线前必须通过)

- [ ] `mysql` healthy

- [ ] `notification-svc` 可访问 `/actuator/health`

```bash

curl http://localhost:8082/actuator/health
curl -u ${RABBITMQ_USER}:${RABBITMQ_PASSWORD} http://localhost:15672/api/overview
```

## 4. 故障与回滚 (单机)

### 4.1 快速止损

```bash
# 停止 Java 新服务,回退到 Python 路径
docker compose -f docker-compose.single8gb.yml stop notification-svc im-gateway-svc
```

### 4.2 重启核心依赖

```bash
docker compose -f docker-compose.single8gb.yml restart mysql rabbitmq
```

### 4.3 常见问题

- 内存高: 降低 JVM `Xmx` 到 `512m`
- 队列堆积: 临时加消费者实例或降频生产
- 磁盘高: 清理日志、压缩历史、检查慢查询

## 5. 扩容动作手册 (从单机到多机)

### 阶段 1: 先拆数据库

- MySQL 迁移至托管 RDS 或独立主机
- 应用仅改 `SPRING_DATASOURCE_URL`
- 验证点: 延迟、连接池、慢查询

### 阶段 2: 再拆消息队列

- RabbitMQ 迁移托管服务或 2+1 拓扑
- 应用仅改 `SPRING_RABBITMQ_HOST`
- 验证点: 可靠投递、DLQ、重试

### 阶段 3: 应用横向扩容

- `notification-svc`、`im-gateway-svc` 各加 1~2 实例
- 接入 Nginx/Ingress
- 验证点: 无状态、幂等、重复消费控制

---

## 6. 持续迭代节奏 (建议)

- 双周发布一次(稳定性周 + 能力周)
- 每周固定 1 次容量评审(CPU/内存/队列/慢查询)
- 每月固定 1 次恢复演练(备份恢复 + 回滚验证)

发布门禁:

- 单测覆盖 >= 80%
- 无 Critical CVE
- 回滚脚本可执行
- 灰度/开关策略可控

---

## 7. 何时必须升级到 16GB+

满足任一条即进入扩容窗口:

- 内存长期 > 85% 且出现 swap
- P99 连续 3 天高于目标
- RabbitMQ 队列深度峰值 > 5000
- 单机月故障次数 > 2
