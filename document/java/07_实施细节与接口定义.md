## 1. 通知事件 API 契约

### 1.1 消息发布接口 (Python 端)

**主题**: `notification.events` (RabbitMQ Topic Exchange)

**消息格式**:
```json
{
  "_schema_version": "1.0",
  "id": "evt_1704067200000",
  "event_type": "paper_trade",
  "channel": "dingtalk",
  "paper_id": 42,
  "trade": {
    "code": "600000",
    "name": "浦发银行",
    "direction": "buy",
    "amount": 100,
    "price": 9.50,
    "trade_date": "2026-01-01",
    "commission": 0.0285,
    "cash_before": 1000000,
    "cash_after": 999050,
    "reason": "keep_increasing 策略命中"
  },
  "timestamp": "2026-01-01T14:30:00Z",
  "retry_count": 0,
  "max_retries": 3
}
```

**Python 生产者** (`quantia/lib/notification_publisher.py`):
```python
import pika
import json
import uuid
import datetime

class NotificationPublisher:
    def __init__(self, rabbitmq_host='localhost', rabbitmq_user='quantia', rabbitmq_pwd='pwd'):
        creds = pika.PlainCredentials(rabbitmq_user, rabbitmq_pwd)
        self.connection = pika.BlockingConnection(
            pika.ConnectionParameters(host=rabbitmq_host, credentials=creds))
        self.channel = self.connection.channel()
        self.channel.exchange_declare(
            exchange='notification',
            exchange_type='topic',
            durable=True)
    
    def send_trade_notification(self, paper_id, trade_result, trade_date):
        """发送模拟交易成交通知"""
        msg = {
            '_schema_version': '1.0',
            'id': f"evt_{int(datetime.datetime.now().timestamp())}",
            'event_type': 'paper_trade',
            'channel': 'dingtalk',
            'paper_id': paper_id,
            'trade': {
                'code': trade_result['code'],
                'name': trade_result['name'],
                'direction': trade_result['direction'],
                'amount': trade_result['amount'],
                'price': float(trade_result['price']),
                'trade_date': str(trade_date),
                'commission': float(trade_result.get('commission', 0)),
                'cash_before': float(trade_result.get('cash_before', 0)),
                'cash_after': float(trade_result.get('cash_after', 0)),
                'reason': trade_result.get('reason', '')
            },
            'timestamp': datetime.datetime.utcnow().isoformat() + 'Z',
            'retry_count': 0,
            'max_retries': 3
        }
        
        self.channel.basic_publish(
            exchange='notification',
            routing_key='notification.paper_trade',
            body=json.dumps(msg),
            properties=pika.BasicProperties(
                content_type='application/json',
                delivery_mode=2))  # persistent
```

### 2.3 IM 指令持久化表

**表名**: `cn_stock_trade_command`

```sql
CREATE TABLE cn_stock_trade_command (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    command_id VARCHAR(64) NOT NULL UNIQUE KEY,
    channel VARCHAR(32) NOT NULL,
    message_id VARCHAR(128),
    operator VARCHAR(128) NOT NULL,
    command_type VARCHAR(32),
    code VARCHAR(16),
    amount INT,
    price_limit DECIMAL(10,4),
    estimated_value DECIMAL(15,2),
    status VARCHAR(32) NOT NULL DEFAULT 'pending',
    wind_control_result VARCHAR(20),
    rejection_reason VARCHAR(500),
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_status (status),
    INDEX idx_created_at (created_at),
    INDEX idx_operator (operator)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**状态机**:
```
pending (初始)
  ↓ 风控检查
approved (通过,等待执行) 或 rejected (拒绝)
  ↓ 执行
executed (已执行) 或 failed (执行失败)
```

## 3. 交易命令事件流 (Kafka)

### 3.1 Topic 设计

**Topic**: `trading.commands`
```yaml
Partitions: 3
```

**消息格式**:
```json
{

  "channel": "dingtalk",
  "operator": "user_123",

  "code": "600000",

  "estimated_value": 1050,
```

### 3.3 Python 消费者示例

```python
from kafka import KafkaConsumer
import json

consumer = KafkaConsumer(
    'trading.commands',
    bootstrap_servers=['kafka:9092'],
    group_id='live-trade-executor',
    value_deserializer=lambda m: json.loads(m.decode('utf-8')),
    auto_offset_reset='earliest',
    enable_auto_commit=False)

for message in consumer:
    cmd = message.value
    print(f"收到交易命令: {cmd['command_id']} - {cmd['code']} {cmd['amount']}股")
    
    # 处理交易命令
    try:
        # 调用 Broker API / live-trade-svc
        result = execute_trade(cmd)
        consumer.commit()  # 手动提交 offset
    except Exception as e:
        # 不提交 offset,重试
        logger.error(f"交易执行失败: {e}")
```

## 4. AI 决策网关接口

### 4.1 评分请求

**Endpoint**: `POST /api/ai-decision/gate`

**请求**:
```json
{
  "request_id": "req_uuid",
  "code": "600000",
  "name": "浦发银行",
  "direction": "buy",
  "source": "strategy:keep_increasing",
  "price": 9.50,
  "volume": 100000000,
  "pe": 8.5,
  "pb": 1.2,
  "roe": 0.15,
  "timestamp": "2026-01-01T14:30:00Z"
}
```

**响应** (200 OK):
```json
{
  "request_id": "req_uuid",
  "code": "600000",
  "score": 72,
  "gate_result": "PASS",
  "model": "gpt-4",
  "latency_ms": 1200,
  "cached": false,
  "reasoning": "公司基本面良好,技术面呈现上升趋势,建议买入"
}
```

**评分规则**:
```
Buy Gate:
  score >= buy_threshold (default 60) → PASS
  score < buy_threshold → REJECT

Sell Gate:
  score <= sell_threshold (default 40) → PASS
  score > sell_threshold → REJECT
```

## 6. 错误处理规范

### 6.1 通用错误响应

```json
{
  "error": "ERROR_CODE",
  "message": "人类可读的错误信息",
  "trace_id": "trace_abc123def456",
  "timestamp": "2026-01-01T14:30:00Z"
}
```

### 6.2 错误码定义

| 错误码 | HTTP 状态 | 含义 | 缓解方案 |
|---|---|---|---|
| `INVALID_REQUEST` | 400 | 请求参数不合法 | 检查参数格式 |
| `WIND_CONTROL_FAILED` | 400 | 风控检查失败 | 降低金额 或 联系管理员 |
| `SERVICE_UNAVAILABLE` | 503 | 服务暂时不可用 | 自动重试 |
| `AI_GATE_TIMEOUT` | 504 | AI 决策超时 | 降级处理(自动通过) |
| `DATABASE_ERROR` | 500 | 数据库错误 | 告警运维,重试 |

## 11. 测试用例框架

### 11.1 单元测试

```java
@SpringBootTest
public class NotificationProcessorTest {
    
    @Test
    public void testSuccessfulNotificationSend() {
        // arrange
        NotificationEvent event = buildTestEvent();
        
        // act
        processor.process(event);
        
        // assert
        assertThat(notificationRepository.findById(event.getId())
            .get().getStatus()).isEqualTo("sent");
    }
    
    @Test
    public void testRetryAfterFailure() {
        // 模拟钉钉 API 失败
        dingtalkClient.setFailureMode(true);
        
        // 第一次调用 process() 应该失败并抛出异常
        assertThrows(DingTalkException.class, () -> processor.process(event));
        
        // 消息应该进入重试队列
        assertThat(event.getRetryCount()).isEqualTo(1);
        assertThat(event.getNextRetryAt()).isNotNull();
    }
}
```

### 11.2 集成测试

```java
@SpringBootTest
@DirtiesContext
public class EndToEndTest {
    
    @Test
    public void testPythonToJavaNotificationFlow() {
```
