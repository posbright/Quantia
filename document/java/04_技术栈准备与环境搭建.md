## 1. 技术栈选型

### 1.1 开发框架

| 组件 | 版本 | 用途 | 备选 |
|---|---|---|---|
| **Spring Boot** | 3.1.0+ | Web 框架 | Quarkus, Micronaut |
| **Spring Cloud** | 2022.0.3+ | 微服务治理 | 自建 (不推荐) |
| **Spring Data JPA** | (包含) | ORM | MyBatis, jOOQ |
| **Spring Cloud Gateway** | (包含) | API 网关 | Kong, nginx |
| **Micrometer Tracing + Zipkin** | (包含) | 链路追踪 | Jaeger, DataDog |
| **Micrometer** | (包含) | 指标收集 | Prometheus 直连 |

### 1.2 消息队列

| 选择 | 用途 | 特点 | 理由 |
|---|---|---|---|
| **RabbitMQ 3.11+** | 通知、异步任务 | AMQP、可靠投递、重试 | ✅ 通知场景最佳 |
| **Kafka 7.4+** | 交易事件流 | 高吞吐、持久化、顺序性 | ✅ 交易事件流最佳 |
| Redis Stream | (备选) | 轻量级 MQ | 可选性能要求不高场景 |

### 1.3 数据存储

| 组件 | 版本 | 用途 |
|---|---|---|
| **MySQL** | 8.0+ | 业务数据库(共享) |
| **Redis** | 7.0+ | 缓存、配置、分布式锁 |
| **Elasticsearch** | 8.0+ | (可选)日志索引 |

### 1.4 监控与可观测性

| 组件 | 版本 | 用途 |
|---|---|---|
| **Prometheus** | 2.40+ | 指标聚合 |
| **Grafana** | 9.5+ | 指标可视化 |
| **Zipkin** | 2.23+ | 链路追踪 |
| **ELK Stack** | 8.0+ | (可选)日志聚合 |

### 2.2 Docker Compose 一键启动

**docker-compose.yml** (放在项目根目录):
```yaml
version: '3.9'

services:
  mysql:
    image: mysql:8.0.33
    container_name: quantia-mysql
    ports:
      - "3306:3306"
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: quantiadb
      MYSQL_USER: quantia
      MYSQL_PASSWORD: quantia_pwd
      TZ: 'Asia/Shanghai'
    volumes:
      - mysql_data:/var/lib/mysql
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5

  rabbitmq:
    image: rabbitmq:3.11-management-alpine
    container_name: quantia-rabbitmq
    ports:
      - "5672:5672"
      - "15672:15672"
    environment:
      RABBITMQ_DEFAULT_USER: quantia
      RABBITMQ_DEFAULT_PASS: quantia_pwd
    volumes:
      - rabbitmq_data:/var/lib/rabbitmq
    healthcheck:
      test: rabbitmq-diagnostics -q ping
      interval: 10s
      timeout: 5s
      retries: 5

  kafka:
    image: confluentinc/cp-kafka:7.4.0
    container_name: quantia-kafka
    ports:
      - "9092:9092"
    environment:
      KAFKA_BROKER_ID: 1
      KAFKA_ZOOKEEPER_CONNECT: zookeeper:2181
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://kafka:29092,PLAINTEXT_HOST://127.0.0.1:9092
      KAFKA_LISTENER_SECURITY_PROTOCOL_MAP: PLAINTEXT:PLAINTEXT,PLAINTEXT_HOST:PLAINTEXT
      KAFKA_INTER_BROKER_LISTENER_NAME: PLAINTEXT
      KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1
    depends_on:
      - zookeeper
    healthcheck:
      test: kafka-broker-api-versions --bootstrap-server localhost:9092
      interval: 10s
      timeout: 5s
      retries: 5

  zookeeper:
    image: confluentinc/cp-zookeeper:7.4.0
    container_name: quantia-zookeeper
    ports:
      - "2181:2181"
    environment:
      ZOOKEEPER_CLIENT_PORT: 2181
      ZOOKEEPER_TICK_TIME: 2000
    volumes:
      - zookeeper_data:/var/lib/zookeeper/data

  zipkin:
    image: openzipkin/zipkin:latest
    container_name: quantia-zipkin
    ports:
      - "9411:9411"
    environment:
      STORAGE_TYPE: mem

  prometheus:
    image: prom/prometheus:latest
    container_name: quantia-prometheus
    ports:
      - "9090:9090"
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml
      - prometheus_data:/prometheus
    command:
      - '--config.file=/etc/prometheus/prometheus.yml'
      - '--storage.tsdb.path=/prometheus'

  grafana:
    image: grafana/grafana:latest
    container_name: quantia-grafana
    ports:
      - "3000:3000"
    environment:
      GF_SECURITY_ADMIN_PASSWORD: admin
      GF_USERS_ALLOW_SIGN_UP: false
    volumes:
      - grafana_data:/var/lib/grafana
    depends_on:
      - prometheus

volumes:
  mysql_data:
  rabbitmq_data:
  zookeeper_data:
  prometheus_data:
  grafana_data:

networks:
  default:
    name: quantia-network
```

**启动命令**:
```bash
# 一键启动所有服务
docker-compose up -d

# 等待各服务就绪(2-3 分钟)
docker-compose ps

# 验证各服务
curl http://localhost:15672  # RabbitMQ UI (user: quantia, pwd: quantia_pwd)
curl http://localhost:9411   # Zipkin
curl http://localhost:9090   # Prometheus
curl http://localhost:3000   # Grafana (admin/admin)
```

## 5. 数据库初始化

```sql
-- 通知事件表
CREATE TABLE IF NOT EXISTS `cn_stock_notification_event` (
    `id` BIGINT AUTO_INCREMENT PRIMARY KEY,
    `dedupe_key` VARCHAR(128) NOT NULL UNIQUE,
    `event_type` VARCHAR(64) NOT NULL,
    `channel` VARCHAR(32) NOT NULL,
    `paper_id` INT DEFAULT NULL,
    `status` VARCHAR(20) NOT NULL DEFAULT 'pending',
    `retry_count` INT NOT NULL DEFAULT 0,
    `max_retries` INT NOT NULL DEFAULT 3,
    `next_retry_at` DATETIME DEFAULT NULL,
    `payload_json` LONGTEXT,
    `response_json` LONGTEXT,
    `error_message` VARCHAR(1000),
    `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    `sent_at` DATETIME DEFAULT NULL,
    INDEX `idx_status_retry` (`status`, `next_retry_at`),
    INDEX `idx_created_at` (`created_at`),
    INDEX `idx_paper_id` (`paper_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 交易命令表
CREATE TABLE IF NOT EXISTS `cn_stock_trade_command` (
    `id` BIGINT AUTO_INCREMENT PRIMARY KEY,
    `command_id` VARCHAR(64) NOT NULL UNIQUE,
    `channel` VARCHAR(32) NOT NULL,
    `message_id` VARCHAR(128),
    `operator` VARCHAR(128),
    `command_type` VARCHAR(32),
    `code` VARCHAR(16),
    `amount` INT,
    `price_limit` DECIMAL(10, 4),
    `status` VARCHAR(32) NOT NULL DEFAULT 'pending',
    `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX `idx_status` (`status`),
    INDEX `idx_created_at` (`created_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- AI 决策配置表
CREATE TABLE IF NOT EXISTS `cn_stock_ai_decision_config` (
    `id` INT AUTO_INCREMENT PRIMARY KEY,
    `config_key` VARCHAR(128) NOT NULL UNIQUE,
    `config_value` LONGTEXT,
    `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
    `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 9. 单机小内存可行性与最低配置

### 9.1 可行性结论

- **PoC/试运行可行**: 单机可承载 `notification-svc` + `im-gateway-svc` + MySQL + RabbitMQ。
- **全量生产不建议**: 若同时启用 Kafka、全量监控、live-trade 高峰流量，单机会成为单点与容量瓶颈。

### 9.2 最低可运行配置 (单机)

| 项目 | 最低配置 | 备注 |
|---|---|---|
| CPU | 4 vCPU | 低峰可运行 |
| 内存 | 8 GB | 需严格限制 JVM 堆与监控组件 |
| 磁盘 | 120 GB SSD | 预留日志与消息堆积空间 |
| 网络 | 100 Mbps | 内网部署可用 |

### 9.3 建议稳态配置 (单机)

| 项目 | 建议配置 | 备注 |
|---|---|---|
| CPU | 8 vCPU | 可承受日常峰值 |
| 内存 | 16 GB | 可同时运行基础监控 |
| 磁盘 | 200 GB SSD | 保证 MySQL + MQ + 日志留存 |

### 9.4 单机部署边界

- 不建议在单机上同时运行 ELK 全套。
- Kafka 在单机仅可用于开发/演示，不满足高可用。
- 生产若包含 live-trade，建议至少 2~3 台或采用托管 MQ/DB。
