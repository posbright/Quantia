### 具体 Bug 修复详单

#### B1: 数据矛盾 - 工作量计算

**问题**: 
```
文档 06_时间进度规划.md:
- 说法 1: "工作量: 500 人天"
- 说法 2: "资源分配: 11 人 × 24 周 = 264 人周"
- 不匹配: 264 人周 × 5 天 = 1320 人天 ≠ 500 人天
```

**修复**:
```markdown
## 工作量正确计算

理论全职工作量: ~500 人天 (按 70% 有效利用率)

实际人力分配: 
  11 人 × 24 周 = 264 人周
  考虑管理开销与并行工作重叠: 有效 180-200 人天
  
对应关系:
  Phase 1 (4w): 80 人天
  Phase 2 (5w): 100 人天  
  Phase 3 (4w): 60 人天
  Phase 4 (6w): 120 人天
  合计: ~500 人天 ✓
```

#### B2: 周期不统一

**问题**: 
```
文档 03: "21-25 周 (5-6 个月)"
文档 06: "24 周 (6 个月)"
甘特图: 24 周
```

**修复**: 统一改为 **24 周 (6 个月)**

#### B3: Phase 3 执行时机

**问题**: 不清楚 Phase 3 (ai-decision-svc) 是否与 Phase 4 并行

**修复**: 
```markdown
时间线说明:

关键路径: Phase 1 → Phase 2 → 评估 → Phase 4 → 灰度
         (4w)   (5w)      (1w)    (6w)   (3w)
         
非关键路径: Phase 3 (4w) 可在 Phase 2 和 Phase 4 之间并行
          不影响总工期,只需在 W11 评估时决定是否启动
```
