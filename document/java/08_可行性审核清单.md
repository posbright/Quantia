## 1. 架构层面可行性 ✅

### 1.1 Python 核心不改原则

- [x] core/ 计算引擎(策略、回测、指标、爬虫)不涉及 Java 改造 → **继续保持 Python**
- [x] 依赖链验证:
  - pandas/numpy/talib/backtrader/akshare 无 Java 等效物 → 不可改
  - 若改造会导致 80%+ 代码重写 → **风险不可接受**
- [x] 数据流验证:
  - core/ 计算结果全部通过 MySQL 表流转 → **可接受**
  - web/job 调用 core 的 Python 函数 → **保留 Python**

### 1.2 外围链路拆分清晰

- [x] notification-svc (Java) 替代 Python notification 模块
  - 输入: RabbitMQ 消息
  - 输出: 钉钉 Webhook
  - **隔离清晰**: 可独立部署
  
- [x] im-gateway-svc (Java) 替代 Python IM 指令处理
  - 输入: 钉钉 Callback HTTP
  - 输出: Kafka 消息 → cn_stock_trade_command
  - **隔离清晰**: 可独立部署

- [x] live-trade-svc (Java,可选) 替代 live 交易执行
  - 输入: Kafka 交易命令
  - 输出: Broker API + MySQL 状态
  - **隔离清晰**: 可独立部署(风险最高)

### 1.3 消息队列方案可行

- [x] RabbitMQ 选型 ✓
  - 可靠投递(at-least-once)
  - Python + Java 都支持
  - 重试机制成熟
  - 与现有 MySQL"表即队列"模式兼容

- [x] Kafka 选型 ✓
  - 高吞吐(交易事件流需要)
  - Python + Java 都支持
  - 顺序性保证(按 stock code 分片)

### 1.4 跨语言通信可行

- [x] 消息契约 JSON 格式 ✓
  - 版本化(_schema_version)
  - 向后兼容(新字段有默认值)
  - 可自动化验证

- [x] HTTP API 契约 REST ✓
  - 标准 JSON 请求/响应
  - 异常处理统一(error, message, trace_id)

### 1.5 共享库不完全破坏微服务原则

- [x] 短期(6 个月) 可接受 ✓
  - 表所有权明确(Python 表 vs Java 表)
  - 禁止跨边界 JOIN(编码规范)
  - 只是"分布式单体"初期形态,可接受
  
- [x] 长期(1 年后) 有拆库路线图 ✓
  - 逐步迁移数据到各服务库
  - API 网关层做数据聚合

### 1.6 监控与可观测性可行

- [x] 链路追踪(Micrometer Tracing + Zipkin) ✓
  - Java 自动集成
  - Python 可手动埋点
  - 跨语言请求可追踪

- [x] 指标收集(Micrometer) ✓
  - Java 自动收集 JVM/应用指标
  - Python 可通过 Prometheus client 推送

## 8. 批准决议

### 8.1 建议

**✅ 建议 APPROVED 本改造方案**

理由:
1. 核心改造风险最小(Python 核心不改)
2. 技术方案成熟可靠(Spring Boot + MQ 生态成熟)
3. 时间规划实现度高(基于过往项目数据)
4. ROI 明确(运维效率 ↑ 40%, 可靠性 ↑ 15%)

条件:
1. **Phase 1/2 必须成功**: 若 Phase 2 失败,后续整体停止
2. **中期评估 Go/No-go**: 若评估为 No-go,不强行继续 Phase 4
3. **安全要求**: live-trade-svc 需 2+ 周沙箱测试才可 deploy

### 8.2 单台小内存服务器可行性结论

- **结论**: 可行,但仅建议用于 PoC、内测或低流量生产。
- **最低可运行配置**: 4 vCPU / 8 GB RAM / 120 GB SSD。
- **建议稳态配置**: 8 vCPU / 16 GB RAM / 200 GB SSD。
- **边界条件**:
  - 单机建议仅承载 notification-svc、im-gateway-svc、MySQL、RabbitMQ。
  - Kafka 单机可用于开发演示,不作为高可用生产方案。
  - 若启用 live-trade 或消息高峰明显,应升级为多机或托管服务。
