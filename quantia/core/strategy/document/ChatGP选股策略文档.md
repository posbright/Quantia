## 2. 股票池构建（五层过滤体系）

### 2.1 第一层：财务安全过滤（排除地雷股）

| 指标 | 条件 | 代码字段 | 实现状态 | 说明 |
|------|------|---------|----------|------|
| 资产负债率 | < 60% | `debt_asset_ratio` | ✅ 已实现 | 排除高财务杠杆。银行/地产天然高杠杆，此条件会排除金融股 |
| 每股经营现金流 | > 0 | `per_netcash_operate` | ✅ 已实现 | 确保经营造血能力 |
| 流动比率 | ≥ 1.0 | `current_ratio` | ✅ 已实现 | 短期偿债能力 |
| 速动比率 | ≥ 0.7 | `speed_ratio` | ✅ 已实现 | 扣除存货后的快速偿债能力 |

> **v3.0 变更**：
> - ~~商誉/净资产 < 20%~~：已移除。东方财富综合选股API不提供商誉数据，无法实现
> - 速动比率从 0.8 降至 **0.7**：A股制造业速动比率普遍偏低，0.8 过于严格
> - 新增流动比率和速动比率的代码实现

### 2.2 第二层：盈利能力筛选（选出优质公司）

| 指标 | 条件 | 代码字段 | 实现状态 | 说明 |
|------|------|---------|----------|------|
| ROE(加权) | ≥ 15% | `roe_weight` | ✅ 已实现 | 股东回报率基准 |
| 毛利率 | ≥ 25% | `sale_gpr` | ✅ 已实现 | 产品定价权指标 |
| 净利率 | ≥ 8% | `sale_npr` | ✅ 已实现 | 成本控制能力 |
| ROA | ≥ 4% | `jroa` | ✅ 已实现 | 资产使用效率 |

> **v3.0 变更**：
> - 毛利率从 30% 降至 **25%**：A股优质制造业（如工程机械、家电龙头）毛利率多在 25-35%，30% 门槛将大量优质工业企业排除
> - 净利率从 10% 降至 **8%**：与毛利率调整配套，工业企业净利率 8-15% 已属优秀
> - ROA 从 5% 降至 **4%**：资产较重的行业（如制造业）ROA 天然偏低，4% 更合理
> - ~~ROIC ≥ 10%~~：已移除。`roic` 字段在数据源中覆盖率不足（大量NULL），作为必选条件会误杀大量优质股票。已移至护城河评分模型中作为加分项使用

> 💡 **优质公司标准**：ROE ≥ 15% 且毛利率 ≥ 25%，说明公司在行业中具有竞争优势

### 2.3 第三层：成长质量筛选（确保可持续增长）

| 指标 | 条件 | 代码字段 | 实现状态 | 说明 |
|------|------|---------|----------|------|
| 营收3年CAGR | > 8% | `income_growthrate_3y` | ✅ 已实现 | 业务规模扩张 |
| 净利润3年CAGR | > 8% | `netprofit_growthrate_3y` | ✅ 已实现 | 盈利能力增长 |
| 扣非净利润增长率 | > 0% | `deduct_netprofit_growthrate` | ✅ 已实现 | 确认主业增长，排除靠一次性损益拉动利润的公司 |

> **v3.0 变更**：
> - 营收和净利润3年CAGR从 10% 降至 **8%**：10% 在经济降速周期过于严格，且稳健蓝筹（如公用事业、消费龙头）3年CAGR通常在 8-12% 区间
> - 新增扣非净利润增长率 > 0% 的代码实现：排除靠非经常性损益粉饰业绩的公司
> - ~~净利润CAGR > 营收CAGR（推荐）~~：已移除。此条件作为硬性过滤不合理——高增长公司前期常有利润率压缩的阶段（如研发投入期），这并不代表质量差

### 2.4 第四层：护城河评估（护城河筛选）

> **定性分析，已通过护城河评分模型（MoatScorer）量化实现。**

| 维度 | 权重 | 评估依据 | 代码实现 |
|------|------|----------|----------|
| 盈利能力 | 25% | ROE、毛利率、净利率 | ✅ `MoatScorer` |
| 稳定性 | 20% | 盈利波动性、分红记录 | ✅ `MoatScorer` |
| 成长能力 | 20% | 营收增长、利润增长 | ✅ `MoatScorer` |
| 经营效率 | 15% | ROA、ROIC、周转率 | ✅ `MoatScorer` |
| 财务安全 | 20% | 负债率、现金流、流动性 | ✅ `MoatScorer` |

**护城河类型识别（AI 辅助）：**

| 类型 | 特征 | 典型行业 | 代码实现 |
|------|------|----------|----------|
| 品牌效应 | 高毛利率 ≥40%，定价权 | 白酒、奢侈品 | ✅ `moat_ai_service.py` |
| 技术壁垒 | 高研发投入，专利护城 | 医药、芯片 | ✅ `moat_ai_service.py` |

| 转换成本 | 高客户粘性，低流失 | 软件、金融 | ✅ `moat_ai_service.py` |
| 牌照壁垒 | 稀缺准入资质 | 银行、券商 | ✅ `moat_ai_service.py` |

> **说明**：护城河评估作为**加分项和参考**，不作为硬性筛选条件。用户可在 Web UI 中配置护城河评分阈值。
> AI 护城河分析需配置 OpenAI 兼容的 LLM API，详见 `moat_ai_service.py`。

### 2.5 第五层：估值约束（买入价格控制）

| 指标 | 条件 | 代码字段 | 实现状态 | 说明 |
|------|------|---------|----------|------|
| PE(TTM) | 0 < PE ≤ 50 | `pe9` | ✅ 已实现 | 排除亏损和泡沫 |
| PB(MRQ) | ≤ 10 | `pbnewmrq` | ✅ 已实现 | 资产估值上限 |
| 股息率 | ≥ 0（推荐） | `zxgxl` | ❌ 仅作加分项 | 有分红意识加分，不作硬性条件 |

> **v3.0 变更**：
> - ~~PE ≤ 5年历史中位数~~：已移除。数据源不提供5年历史PE分位数据，无法实现。需要额外数据维护成本大
> - ~~PEG ≤ 1.5~~：已移除。`ycpeg` 为东财预测值，覆盖率不稳定且基于分析师预测（可靠性存疑）。PEG 需要稳定的利润增速预测，作为必选条件风险较大
> - 新增 PB ≤ 10 的实现：排除极端高估值股票
> - 股息率不作硬性条件：A股大量优质成长股不分红或分红很少（如科技股），硬性要求 >0 会误杀

👉 **最终股票池规模预期**：30–80 只（条件放宽后覆盖面更合理）

## 4. 技术择时策略（三类买入信号）

> **原则**：基本面决定方向，技术指标仅用于择时。以下三类策略已在代码中实现。

### 4.1 第一类：趋势回调买入（核心）— `trend_pullback`

**适用场景**：优质公司，长期趋势未破，回调到均线支撑位

| 条件 | 参数 | 代码实现 |
|------|------|----------|
| 均线多头排列 | MA20 > MA60 | ✅ `value_invest_strategies.py` |
| 价格回调至均线 | 价格在 MA20 ±3% 范围内 | ✅ |
| RSI 温和区域 | RSI(14) ∈ [35, 55] | ✅ |
| 回调缩量确认 | 成交量 < MA20 × 0.8 | ✅ |

**动作** → 买入 30% 目标仓位

### 4.2 第二类：超跌反弹买入（情绪错杀）— `oversold_rebound`

**适用场景**：市场恐慌但基本面未变，技术面超卖

| 条件 | 参数 | 代码实现 |
|------|------|----------|
| RSI 超卖 | RSI(14) < 30 | ✅ `value_invest_strategies.py` |
| 跌至布林下轨附近 | 价格 ≤ 布林下轨 × 1.02 | ✅ |
| 反弹确认 | 收复布林下轨 + 阳线 | ✅ |
| 放量确认 | 成交量 > MA5 × 1.2 | ✅ |

**动作** → 买入 20%–30% 仓位

### 4.3 第三类：突破确认买入（谨慎）— `breakout_confirm`

**适用场景**：横盘整理后放量突破，只用于核心资产

| 条件 | 参数 | 代码实现 |
|------|------|----------|
| 横盘整理 | 40日振幅 < 25% | ✅ `value_invest_strategies.py` |
| 创新高 | 创40日最高价 | ✅ |
| 放量突破 | 成交量 ≥ MA20 × 1.5 | ✅ |
| 涨幅确认 | 当日涨幅 > 2% | ✅ |
| 均线支撑 | 收盘价 > MA60 | ✅ |

**动作** → 小仓位试探（≤20%）

### 4.4 BOLL 指标辅助参考（新增）

> **定位**：BOLL 作为辅助确认指标，不单独触发买入/卖出，需与趋势和成交量信号组合使用。

| 场景 | BOLL 参考信号 | 使用建议 |
|------|---------------|----------|
| 趋势回调买入 | 回调到中轨附近并企稳，中轨保持上行 | 可增强回调买点置信度 |
| 超跌反弹买入 | 跌破下轨后 1-3 日重新收回下轨 | 与 RSI<30、放量反包共同确认 |
| 突破确认买入 | 上穿上轨且带量，随后不快速跌回轨道内 | 防止假突破，优先右侧小仓试探 |
| 技术性减仓 | 连续贴上轨但成交量放大、涨幅放缓 | 作为分批减仓触发参考 |
| 趋势破坏卖出 | 跌破中轨后反抽不过，且轨道开始走平/向下 | 与 MA60 破位信号叠加确认 |

**默认参数建议**：`BOLL(20, 2)`，与现有技术策略保持一致口径。

## 5. 卖出策略（比买入更重要）

### 5.1 技术性减仓（估值未崩，但涨过头）

| 条件（任一满足） | 说明 |
|------------------|------|
| RSI > 75 且连续 3 日 | 严重超买 |
| 价格触及布林上轨并放量滞涨 | 上方压力大 |

**动作** → 减仓 20%–30%

### 5.2 趋势破坏卖出（必须执行）

| 条件（全部满足） | 说明 |
|------------------|------|
| 周线跌破 MA60 | 长期趋势转空 |
| 连续 2 周无法收回 | 确认非假跌破 |
| 下跌放量 | 抛压真实 |

**动作** → 清仓

### 5.3 基本面卖出（优先级最高）

- 财务质量恶化（ROE 连续下滑、现金流转负）
- 核心竞争力下降（市占率丢失、毛利率持续下降）
- 行业长期逻辑被证伪

👉 **不看技术，直接卖**

## 7. 风控规则（系统级）

### 7.1 回撤控制（组合层面）

| 触发条件 | 动作 |
|---------|------|
| 组合最大回撤 > 15% | 停止加仓 |
| 组合最大回撤 > 20% | 降低整体仓位至 50% |

### 7.2 黑天鹅保护

| 触发条件 | 动作 |
|---------|------|
| 个股单日跌幅 > 8%，且无明确利好解释 | 临时冻结交易，次日观察 |
| 两日累计跌幅 > 12% | 强制减仓，降至半仓 |

## 9. 执行流程

### 每日自动执行（收盘后）

```
Phase 1: 数据获取（API调用）
  ├─ init_job（数据库初始化）
  ├─ fetch_data_job（K线缓存增量更新）
  ├─ basic_data_daily_job（实时行情入库）
  ├─ selection_data_daily_job（综合选股数据入库 → cn_stock_selection）
  ├─ basic_data_other_daily_job（龙虎榜、大宗交易等）
  └─ basic_data_after_close_daily_job（收盘后数据）

Phase 2: 数据分析（本地计算，无API调用）
  ├─ gpt_value_data_job（基本面筛选 → cn_stock_strategy_gpt_value）
  ├─ streaming_analysis_job（技术指标+K线形态+策略择时，流式处理）
  └─ backtest_data_daily_job（策略回测验证）
```

### 手动补跑

```bash
# 补跑基本面筛选
python3 quantia/job/gpt_value_data_job.py 2026-02-14

# 补跑技术择时
python3 quantia/job/strategy_data_daily_job.py 2026-02-14

# 补跑回测
python3 quantia/job/backtest_data_daily_job.py
```

## 10. 策略核心总结

> **基本面决定方向，技术指标决定时机，风控决定你能走多远。**

### 与 v2.0 相比的关键变更

| 变更项 | v2.0 | v3.0 | 优化理由 |
|--------|------|------|----------|
| 毛利率 | ≥ 30% | ≥ 25% | 覆盖优质制造业（工程机械、家电龙头） |
| 净利率 | ≥ 10% | ≥ 8% | 配套毛利率调整，工业企业 8%+ 已属优秀 |
| 营收/利润CAGR | > 10% | > 8% | 适应经济降速周期，覆盖稳健蓝筹 |
| ROA | ≥ 5% | ≥ 4% | 资产较重行业天然偏低 |
| 速动比率 | ≥ 0.8 | ≥ 0.7 | A股制造业普遍偏低 |
| 商誉/净资产 | < 20% | 移除 | 数据源不提供 |
| ROIC | ≥ 10% | 移至评分体系 | 数据覆盖率不足，不宜作硬性条件 |
| PEG | ≤ 1.5 | 移除 | 基于分析师预测，可靠性存疑 |
| PE历史中位数 | ≤ 5年中位 | 移除 | 需额外维护历史PE数据 |
| 股息率 | > 0 | 仅作加分项 | 成长股普遍不分红，不宜硬性要求 |
| 新增流动比率 | — | ≥ 1.0 | 补充偿债能力评估 |
| 新增扣非利润 | — | > 0% | 排除靠非经常性损益粉饰的公司 |
| 新增 PB | — | ≤ 10 | 排除极端高估值 |
| 新增 ROA | — | ≥ 4% | 新增资产效率评估 |
| 技术择时 | 口语化描述 | 对齐代码参数 | 与 `value_invest_strategies.py` 一一对应 |

## 11. 筛选界面与可视化增强需求（新增）

### 11.1 筛选结果页交互

- 在“筛选结果列表”中，股票代码列支持点击。
- 点击股票代码后，进入“股票分析详情页”（或右侧抽屉弹层），展示该股票的多周期图表和技术指标。
- 在每行操作区增加“查看回测”按钮，点击后跳转到回测页面并自动带入当前股票代码。

### 11.2 多周期 K 线与技术指标展示

**展示周期（至少）**：

- 日线：1日、5日、10日、15日
- 中周期：月线、季度线
- 长周期：年线

**图表内容**：

- K线图（开高低收 + 成交量）
- 均线：MA5 / MA10 / MA20 / MA60
- BOLL：上轨 / 中轨 / 下轨
- 动量指标：RSI（建议14）
- 可选：MACD（用于趋势强弱确认）

**数据来源**：基于历史数据缓存（`quantia/cache/hist/`）和现有指标计算结果表，避免每次打开详情都实时拉取外部 API。

### 11.3 回测页面联动

点击“查看回测”后，展示该股票在不同时间窗口的策略表现，建议至少包含：

- 近1个月收益率
- 近3个月收益率
- 近6个月收益率
- 近1年收益率
- 年化收益率（可选）
- 最大回撤（可选）
- 胜率（可选）

### 11.4 落地实施建议

1. 先实现“股票代码可点击 + K线详情页”作为第一阶段。
2. 第二阶段增加“查看回测”跳转和默认时间窗口收益展示。
3. 第三阶段补充高级指标（最大回撤、夏普、胜率）和多策略对比。
