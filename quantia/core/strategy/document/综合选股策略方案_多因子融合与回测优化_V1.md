# Quantia 综合选股策略方案（多因子融合与回测优化）

版本：V1.0  
日期：2026-07-10  
目标：在不追求每日出手的前提下，最大化长期风险调整收益（夏普）并提升胜率

---

## 1. 目标与约束

### 1.1 目标
- 构建一个可实盘落地的综合选股策略，融合：
  - 现有策略信号（趋势、突破、低波动、回调等）
  - 基本面质量与估值
  - 筹码结构信息
  - 资金流向信息
- 以组合层面的风险收益为核心，优先优化：
  1. 夏普比率
  2. 胜率
  3. 年化收益
- 允许低频交易，不强求每日有标的。

### 1.2 约束
- 必须基于当前项目已有数据表与现有回测框架，不依赖不可获得字段。
- 参数优化必须采用历史回测，避免仅凭经验设值。
- 需控制最大回撤和交易拥挤，避免策略仅在少数行情有效。

---

## 2. 现有策略盘点（策略选股体系）

当前项目中，技术/形态策略主要包括：
- 放量上涨
- 均线多头
- 低位均线粘合
- 停机坪
- 回踩年线
- 突破平台
- 无大幅回撤
- 海龟交易法则
- 高而窄的旗形
- 放量跌停
- 低ATR成长
- 趋势回调
- 超跌反弹
- 突破确认

独立基本面方向策略包括：
- GPT综合选股
- 基本面选股（fundamental_buy）

策略共性优点：
- 技术类策略对短中期趋势与交易行为响应快。
- 基本面类策略有助于提升中长期稳定性与抗回撤能力。
- 低波动与回撤类策略适合做风险约束层。

策略共性不足：
- 单策略通常偏行情依赖，跨周期稳定性不足。
- 缺少跨策略统一评分和组合层仓位联动。
- 资金流、筹码与基本面尚未形成系统融合框架。

## 3. 可用数据能力边界（按当前项目）

### 3.1 基本面与估值数据
- 主要表：
  - cn_stock_selection（综合快照，含估值、成长、盈利、质量、技术标签、部分资金流标签）
  - cn_stock_financial（历史财务数据，季报维度）
- 可用关键字段（示例）：
  - 估值：pe9, pbnewmrq, ps9, enterprise_value_multiple
  - 盈利质量：roe_weight, sale_gpr, sale_npr, jroa
  - 成长：income_growthrate_3y, netprofit_growthrate_3y, deduct_netprofit_growthrate
  - 偿债/安全：debt_asset_ratio, current_ratio, speed_ratio, per_netcash_operate

### 3.2 筹码数据
- 主要表：
  - cn_stock_chip_distribution
  - cn_stock_chip_race_open
  - cn_stock_chip_race_end
- 可用方向：
  - 成本带聚集度与筹码集中变化
  - 早盘/尾盘抢筹特征
  - 成本带与现价位置关系（套牢盘/获利盘结构）

### 3.3 资金流数据
- 主要来源：
  - cn_stock_selection 内置资金流相关标签字段（如 low_funds_inflow, high_funds_outflow）
  - 部分交易行为标签与量价字段联动
- 可用方向：
  - 低位净流入确认
  - 高位净流出规避
  - 与突破/回调信号共振过滤

### 3.4 行情与指标、回测数据
- 行情与指标：cn_stock_spot, cn_stock_indicators, 本地 hist 缓存
- 回测结果：cn_stock_backtest_portfolio（含 result_json 与绩效指标）
- 当前能力可支持：
  - 单策略回测
  - 组合策略回测
  - 分段验证和参数寻优

结论：当前数据已足以支撑“基本面 + 技术 + 筹码 + 资金流”多因子融合策略。

## 4. 综合策略设计（推荐方案）

### 4.1 总体架构（四层）

1) 股票池层（Quality Universe）  
- 目标：先过滤掉财务脆弱、流动性差、风险高标的。  
- 规则建议：
  - 财务安全：debt_asset_ratio, current_ratio, speed_ratio, per_netcash_operate
  - 质量成长：roe_weight, sale_gpr, sale_npr, income_growthrate_3y, netprofit_growthrate_3y
  - 估值约束：pe9, pbnewmrq（去掉极端高估值）
  - 交易约束：最小成交额、上市天数

2) 信号评分层（Multi-Alpha Score）  
- 将信号拆成四组并标准化打分：
  - 基本面因子组：质量、成长、估值
  - 技术行为组：趋势、突破、回调、低波动
  - 筹码结构组：集中度、成本带、抢筹变化
  - 资金流组：低位流入、高位流出、量价确认
- 组合总分：
  - score_total = w_fund * score_fund + w_tech * score_tech + w_chip * score_chip + w_flow * score_flow

3) 交易执行层（Entry/Exit + Position）  
- 入场：
  - score_total >= 阈值
  - 同时满足至少一类主触发（趋势突破或低波动成长）
  - 且风险过滤通过（非高位流出、非极端回撤破位）
- 出场：
  - 固定止损 + 跟踪止盈 + 持有期上限
  - 信号失效退出（总分跌破退出阈值）

4) 组合风控层（Portfolio Risk）  
- 最大持仓数
- 单票最大权重
- 行业集中度上限
- 市场状态过滤（弱市降低仓位/提高阈值）

### 4.2 候选组合逻辑（融合现有策略优点）

主信号（择时核心）：
- 趋势突破类：突破平台 + 海龟 + 突破确认
- 趋势回调类：回踩年线 + 趋势回调
- 低波稳进类：低ATR成长 + 无大幅回撤 + 均线多头

辅助过滤（提高胜率）：
- 基本面过滤：GPT综合选股条件（放宽为评分化，不做过硬阈值）
- 筹码过滤：筹码集中改善 + 抢筹信号
- 资金过滤：低位流入增强，高位流出减分/禁入

建议组合机制：
- 双门槛：
  - 门槛A（质量门槛）：基本面评分必须达到最低值
  - 门槛B（交易门槛）：技术/筹码/资金综合评分达到入场阈值
- 这样可显著降低“纯技术噪声交易”。

## 5. 参数优化方向与方法

### 5.1 优化参数集合

A. 股票池参数
- 财务安全阈值（负债率、流动性）
- 估值上下限（PE/PB）
- 最小成交额、最小上市天数

B. 信号权重参数
- w_fund, w_tech, w_chip, w_flow
- 主/辅信号权重（突破、回调、低波动）

C. 交易参数
- 入场阈值、退出阈值
- 止损、止盈、跟踪止盈
- 最大持有天数
- 最大持仓数、单票权重

### 5.2 优化目标函数（建议）

使用多目标合成分数：
- objective = a * Sharpe + b * WinRate + c * AnnualReturn - d * MaxDrawdown - e * TurnoverPenalty

建议初始权重：
- a=0.45, b=0.25, c=0.20, d=0.10, e=0.05

> ⚠ 修订(V1.1)：以上公式存在量纲 Bug，直接加权会让“夏普优先”失效。
> 依据：`quantia/core/backtest/risk_metrics.py` 返回的 `sharpe_ratio` 是原始比值（约 0–3），
> 而 `annual_return`、`max_drawdown`、`daily_win_rate` 均为百分数（0–100）。代入建议权重时
> b·WinRate≈12.5、c·AnnualReturn≈4.8 会把 a·Sharpe≈0.63 淹没约 20 倍，寻优实际在最大化胜率数值。
> 正确做法：加权前先对每个指标做同量纲归一化（z-score 或 0–1 缩放），或统一把百分数转成小数后再加权。

> ⚠ 修订(V1.1)：胜率口径必须明确为“逐笔交易胜率”，而非按日胜率。
> 依据：`risk_metrics.py` 同时给出 `daily_win_rate`(按日) 与逐笔配对盈亏(trade_win/trade_loss)。
> 低频策略大部分交易日空仓，按日胜率没有意义，应使用逐笔交易胜率。

并设置硬约束：
- MaxDrawdown <= 上限
- 年交易次数 >= 最低活跃度
- 月度胜率不能长期低于阈值
- 最少交易笔数 >= N（低频下笔数过少时夏普/胜率统计噪声极大、极易过拟合，需强制样本量下限）

### 5.3 回测与验证流程

1) 训练期参数搜索（粗网格 -> 精细网格）
2) 验证期锁定参数评估
3) 滚动窗口 Walk-Forward（核心）
4) 稳健性测试：
- 市场分段（上涨/震荡/下跌）
- 行业分段
- 单因子移除（Ablation）

目标不是单一期间最高收益，而是跨周期稳定高夏普与较高胜率。

## 6. 可行性审计与关键修订点

本方案总体可行，但不能直接按“所有数据全历史可用”的理想状态实现。当前项目中，基本面、技术行情、筹码、资金流的数据覆盖周期与可靠性不同，如果不做数据时点约束和缺失降级，回测结果会虚高或直接无法运行。因此后续实现必须把以下修订点作为硬性前置条件。

### 6.1 历史数据覆盖不一致：必须支持降级模式

依据：
- `cn_stock_financial` 是较完整的长周期财务源，按 `report_date` 季度组织。
- `cn_stock_selection` 属于综合选股快照，更适合当前截面与近期验证；它包含估值、盈利、成长、股东户数、资金流标签、技术标签等，但不应假设其拥有完整长周期日频历史。
- 筹码需拆分为两类，不能一并当作“无历史”：
  - `cn_stock_chip_distribution`（获利比例/集中度/成本带）是**基于本地 K 线缓存 + CYQ 模型估算**（见 `quantia/core/kline/chip_distribution.py`、`cyq.py`），且有历史回填脚本 `quantia/job/backfill_chip_distribution.py`（零 API、纯本地）→ **可逐日重算、可回测**，不应被降级。
  - `cn_stock_chip_race_open/end`（抢筹）为盘中集合竞价快照，`stock_chip_race.py` 在非竞价时段返回 HTTP 462 → **实时性数据、无法历史回测**，只能作为上线后实时增强。

风险：
- 如果三年或五年回测直接强依赖 `selection/chip/flow` 的日频历史，早期日期可能大量为空，导致选不出股票、评分全为 NaN，或因空值处理不当产生异常。
- 如果将缺失数据当成 0 分，会系统性惩罚历史较早阶段，导致回测结果失真。

修订要求：
- 引入 `feature_available_mask`，逐日标记每类数据是否可用。
- 对不可用维度采用中性分或权重归一化，而不是简单记 0。
- 长周期回测默认使用“基本面 + 技术”模式；当筹码/资金流在该日期可用时，再启用增强评分。

建议降级逻辑：

```text
if chip_data_missing:
    w_chip = 0
if flow_data_missing:
    w_flow = 0
normalize(w_fund, w_tech, w_chip, w_flow)
```

### 6.2 财务数据存在前视偏差风险：必须做 PIT 对齐

依据：
- `cn_stock_financial` 的核心日期字段为 `report_date`，代表报告期，不等于市场实际可见日期。
- 年报、季报存在披露滞后。如果回测在报告期刚结束后立即使用财务指标，会产生 Lookahead Bias。

风险：
- 例如在 2024-01 使用 2023-12-31 年报 ROE/利润增速，会用到当时市场尚未公开的数据。
- 这会显著抬高夏普、胜率和年化收益，导致实盘不可复现。

修订要求：
- 所有来自 `cn_stock_financial` 的字段必须按可见日对齐。
- 在没有真实披露日字段时，先采用保守披露滞后规则：
  - Q1/Q2/Q3：`visible_date = report_date + 60自然日`
  - 年报：`visible_date = report_date + 120自然日`
- 每个交易日只能使用 `visible_date <= trade_date` 的最新一条财务记录。

### 6.3 全市场多因子日频回测性能风险：必须先缩小候选池

依据：
- 当前回测数据加载依赖本地 K 线缓存与 `data_feed`。
- 全市场日频逐日计算技术、筹码、资金流、基本面组合评分，在参数寻优时会成倍放大计算量。

风险：
- 直接对全市场 4000+ 股票做全参数网格搜索，容易出现运行极慢或内存占用过高。
- 组合策略参数比单策略更多，过宽网格会带来组合爆炸。

> ⚠ 修订(V1.1)：候选池初筛**不得使用 `dynamic_universe`**做历史回测，否则前视。
> 依据：`quantia/core/composite/dynamic_universe.py` 的 `_query_universe_from_db` 用
> `WHERE date = (SELECT MAX(date) FROM cn_stock_selection)`，永远取最新快照；`fundamentals_signal`
> 里 `fetch_universe(top_n=10000)` 同理。历史回测日若用它初筛，等于把“今天的基本面”用到过去，虚高不可复现。
> 正确做法：历史候选池初筛必须走 `cn_stock_financial` 的 PIT 面板（项目已有验证实现
> `selection_factor_validation_job` 的 `load_financial_panel + lag60`）。

修订要求：
- 先用月度或季度级基本面初筛，将候选池压缩到 100-300 只（初筛数据源须为 PIT 面板，非最新快照）。
- 日频技术/筹码/资金流计算只在候选池内运行。
- 参数搜索采用两阶段：粗网格锁定范围，再小范围精细搜索。

### 6.4 资金流与筹码不能作为主信号，只能作为确认/风险过滤

依据：
- `cn_stock_selection` 里的 `low_funds_inflow`、`high_funds_outflow` 等字段更像离散标签，适合作为确认信息。
- 筹码分布字段依赖抓取与历史沉淀，短期有效性需要独立验证。

风险：
- 如果把资金流或筹码作为主触发信号，可能导致样本覆盖不足、信号跳变大、回测不稳定。
- 这类数据更容易出现短期噪声，单独使用时胜率未必稳定。

修订要求：
- 主信号仍由“基本面质量 + 技术趋势/低波动/突破”产生。
- 资金流用于增强或否决：低位流入加分，高位流出禁入或减分。
- 筹码用于确认：集中度改善、成本带支撑、早盘/尾盘抢筹只提升置信度，不单独触发买入。

### 6.5 新策略必须注册表结构与任务入口

依据：
- 现有日常策略通过 `TABLE_CN_STOCK_STRATEGIES` 统一注册表名、中文名、函数与输出列。
- 已存在表会通过日任务做缺列迁移，但新策略仍需要基础表结构声明。

风险：
- 如果只新增 `composite_alpha_v1.py`，但不注册 `cn_stock_strategy_composite_alpha_v1`，每日任务不会自动建表和写入结果。
- Web 侧策略结果、历史选中标注、参数配置也无法统一展示。

> ⚠ 修订(V1.1)：**不能**把综合评分策略做成 `TABLE_CN_STOCK_STRATEGIES` 的 `check()`——架构不支持。
> 依据：`quantia/job/strategy_data_daily_job.py` 的 `run_check` 以
> `strategy_fun(k, stocks[k], date=date, ...)` **逐票、只传该股单只 K 线**调用，每次只能看到一只股票，
> 无法做 `rank(pct=True)` 这类跨股票横截面打分（本方案 4.1/4.2 的核心）。
> 正确入口：综合评分须做成**独立截面 job**，参考已有 `quantia/job/gpt_value_data_job.py`、
> `quantia/job/selection_score_job.py`（读整张截面表→统一打分→落库）。

修订要求：
- 新建独立截面作业 `composite_alpha_data_job.py`（读截面表→打分→落库），而不是注册进 `TABLE_CN_STOCK_STRATEGIES` 的逐票 `check()`。
- 在 `tablestructure.py` 中新增结果表结构 `cn_stock_strategy_composite_alpha_v1` 及输出列 `_composite_alpha_columns`（供落库/展示/缺列自愈）。
- 在参数配置中新增 `composite_alpha_v1` schema。
- 在该独立作业的调度入口（execute/fetch daily job）中挂载，而非每日逐票策略白名单。

### 6.6 修订后的可执行结论

该综合策略可行，但正确实现路径不是“一次性堆叠所有因子”，而是：
1. 基本面质量先筛池，避免技术策略在低质量股票上反复交易。
2. 技术信号负责触发，优先使用低ATR、均线多头、突破平台、回踩年线等已验证逻辑。
3. 筹码与资金流作为确认层，缺失时自动降级，不影响长周期回测连续性。
4. 财务数据必须使用 PIT 可见日，禁止直接按 `report_date` 前视读取。
5. 回测优化采用 Walk-Forward，而不是单一全样本最优参数。

| 4 | 筹码结论写反：被误判为无历史 | `chip_distribution.py`/`cyq.py` 从 K 线估算，且有 `backfill_chip_distribution.py` | 筹码分布可回测；仅“抢筹”不可回测，见 6.1 |
| 5 | 低频评估失真：全额 NAV 日频夏普 | `risk_metrics` 按含空仓日的整条 NAV 年化(245日) | 明确逐笔胜率、设最少笔数、优先基准相对/持仓期口径，见 5.2 |

优先级：1（架构）> 2（目标函数）> 3（前视）> 4（数据利用）> 5（评估口径）。前三项若不修，实现会直接卡死或产出不可实盘复现的虚高结果。

## 8. 方案核心结论

1) 仅靠单一技术策略难以稳定提升夏普，必须引入基本面质量门槛。  
2) 筹码与资金流应作为交易确认层，而不是单独主信号。  
3) 采用“质量门槛 + 交易门槛”的双门槛机制，可显著提升胜率与一致性。  
4) 参数优化必须用滚动回测和样本外验证，避免过拟合。  
5) 该方案可基于项目现有数据和代码框架实现，但长周期回测必须内置数据缺失降级、财务 PIT 对齐和候选池预筛，不能假设所有维度在全历史可用。

## 10. 实现落地与回测寻优实测结论（V1.1 实证）

### 10.1 已实现产物
- 截面打分模块：`quantia/core/strategy/composite_alpha_v1.py`（横截面打分、维度缺失自动降级重归一、四维度统一 0–100 同量纲后加权、双门槛选股，纯函数可单测）。
- 结果表注册：`tablestructure.py` 新增 `cn_stock_strategy_composite_alpha_v1`（独立表，非逐票策略）。
- 单元测试：`tests/test_composite_alpha_v1.py`（9 项，覆盖降级/同量纲/双门槛/可空出手），全部通过。
- 寻优+验证脚本：`tools/optimize_composite_alpha.py`。已落地 V1.1 所有修订：PIT 财务面板(lag60)、目标函数先同量纲归一化再加权、逐笔交易胜率、多折 Walk-Forward、真实交易成本（买 0.05% / 卖 0.15%）、大盘择时过滤、成本敏感性。

### 10.2 关键实测结论（2022-01~2025-12，290 只，4 折样本外）
1. **交易成本是硬约束**：月度全额换仓下，加入约 0.2% 往返成本后，收益/夏普显著下降，零成本回测严重乐观（务必带成本评估）。
2. **严格多折 Walk-Forward 揭示 regime 依赖**：纯 fund+tech 月度轮动，跨折聚合夏普为负（≈ -0.32），仅 2025 那一折为正；单折验证的高夏普是幸存者式乐观，不能作为结论。
3. **大盘择时（沪深300 低于自身均线则空仓）显著改善风险**：最大回撤从 ≈23% 腰斩到 ≈10%，聚合夏普升到 ≈ -0.13，最优组合零成本夏普转正(+0.06)、含成本约打平。
4. **门槛上探后可绑定**：quality_gate=70 被选为最优，说明质量门槛真正生效并有边际帮助。

### 10.3 暂定稳健默认（尚未达生产标准）
最稳健（最低回撤）组合：`w_fund=0.5, quality_gate=70, trade_gate=60, top_positions=15, regime_ma=60`。
但**不建议直接固化为生产默认**：在 2022–2025 弱市样本上，净成本后整体仅打平、逐笔胜率约 45%（低于目标）。择时把风控做对了，但选股 alpha 尚不足以在弱市产出稳定正的风险调整收益。

### 10.4 有依据的下一步（提升 alpha 的方向）
1. **引入估值因子与 IC 加权**：仓库因子验证（M0/M1，见 selection_scoring 与 selection_factor_validation_job）显示估值反转(pe/pb 低)、营收增速(toi_yoy_ratio) 是最强因子；当前回测 fund 维度是质量+成长等权，缺估值维度，应补入并用 IC 加权而非等权。
2. **修正股票池偏差**：当前回测用财务面板前 N 个代码（近似字母序），非真实指数或基本面最优池，存在选样偏差；应改用明确的指数成分或 PIT 基本面 TopN。
3. **考虑基准相对/对冲口径**：长多头难以战胜 3 年弱市 beta，可评估超额收益(信息比率)口径或指数对冲，剥离 beta 后再看 alpha。
4. **持有期与调仓频率**：月度换仓成本高，可对比季度调仓 + 更长持有，降低换手对夏普的侵蚀。

> 结论：实现与回测框架已就绪且方法学正确（成本、PIT、多折、择时、同量纲目标）。当前瓶颈不在工程实现，而在**选股因子 alpha 不足**；按 10.4 补估值/IC 加权 + 修正股票池后重跑，是把夏普/胜率真正带正的关键路径。

## 11. 补估值维度 + IC 加权后的实证突破（V1.2）

### 11.1 已落地增量
- 打分模块 `composite_alpha_v1.py`：新增 `value` 维度（PE/PB，越低越好）；`_dimension_score`/`compute_composite_scores` 支持 `factor_weights` 做维度内**按因子 IC 加权**（向后兼容，默认等权）。单测扩到 11 项全过。
- 优化脚本 `optimize_composite_alpha.py`：
  - 用 PIT 的 `eps/bps` + 现价构造 `pe_proxy=现价/eps`、`pb_proxy=现价/bps`（防前视）。
  - 新增 `_compute_ic_weights`：在**第一折训练**上按各因子 rank-IC(方向化 vs 前向收益) 加权，避免用评估折未来信息。
  - 股票池由“字母序前 N”改为**随机代表性抽样**（`np.random`，减少选样偏差）。
  - 维度权重网格加入 `w_value`（tech = 1 − w_fund − w_value）。

### 11.2 实证结果（2022-01~2025-12，291 只，4 折样本外，含成本）
| 指标 | 补估值前(V1.1) | 补估值+IC后(V1.2) |
|---|---|---|
| 聚合夏普 | ≈ -0.13 | **0.71** |
| 年化收益 | ≈ 0% | **11.2%** |
| 最大回撤 | ≈ 10% | **8.5%** |
| 逐笔胜率 | ≈ 45% | **53.8%** |
| 最差折夏普 | 负 | **+0.40（4 折全正）** |
| 2× 成本下夏普 | 负 | **0.58** |

- **IC 实证印证仓库 M0 结论**：训练折因子 IC 排序 `atr_pct=0.112`（低波）> `pb_proxy=0.095`、`pe_proxy=0.040`（估值反转）> 成长(toi/netprofit_yoy≈0.01) > `roe_weight≈0`（盈利能力弱），与 selection_scoring/M0 一致。
- 稳健项(obj-e=最差折夏普)生效：无择时但含单折暴涨(年化 103%、min 折为负)的组合被正确排在稳健组合之后。

### 11.3 可上线候选默认（跨折稳健，建议先小仓实盘观察）
```
w_fund=0.40, w_value=0.35, w_tech=0.25,
quality_gate=55, trade_gate=60, top_positions=15,
regime_ma=60（沪深300 低于60日均线则空仓）,
rebalance=21（月度）, lag_days=60（PIT）,
cost_buy=0.0005, cost_sell=0.0015,
维度内因子权重=训练折 IC 加权（估值/低波主导）
```
该组合 4 折样本外夏普全为正、回撤 8.5%、逐笔胜率 53.8%、对成本不敏感，已具备进入小规模实盘观察的条件。

### 11.4 仍需注意 / 后续
- 股票池仍为财务面板随机 291 只，非真实指数 PIT 成分；上真实指数成分后需复验。
- IC 加权仅用第一折估计并固定；可扩展为每折滚动重估（严格 OOS），进一步防漂移。
- 结果落盘：`quantia/cache/optimize/composite_alpha_best_params.json`（含成本敏感性与各折夏普）。
